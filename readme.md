*/ Musnews - это персональная музыкальная платформа, на которую можно загружать свои собственные аудиофайлы, систематизировать их в плейлисты и воспроизводить непосредственно в браузере. Боковая панель новостей живой музыки позволяет вам быть на связи с последними релизами и событиями во время прослушивания.

В проекте Musnews был реализован набор функций для работы с музыкой:

    - аутентификация и управление пользователями: позволяет пользователям регистрироваться и входить в приложение с управляемыми ролями и разрешениями для защищенных маршрутов доступа;
    - комплексное управление музыкальными треками: поддерживает операции CRUD (создание, чтение, обновление, удаление) для музыкальных треков, позволяя добавлять, получать, обновлять и удалять данные треков, а также напрямую передавать файлы .mp3 с сервера;
    - гибкая функциональность плейлистов: пользователи могут создавать и управлять личными плейлистами, организуя музыкальные треки в настраиваемые коллекции. Соответствующие API обрабатываются файлом PlaylistsController.cs;
    - надежная модель данных: использует хорошо структурированную модель данных для пользователей, треков и плейлистов, а также объекты передачи данных (DTO) для удобного взаимодействия с API;
    - сохранение данных через Entity Framework Core: Управление взаимодействием с базой данных с помощью Entity Framework Core, включая миграции для беспрепятственной эволюции схемы базы данных и репозитории для инкапсулированных операций с данными;
    - интерактивный веб-интерфейс пользователя: Базовый веб-интерфейс, созданный с использованием Razor Pages, HTML, CSS и JavaScript, дополненный Bootstrap и jQuery для адаптивного и динамичного пользовательского опыта.

Musnews построен на основе многоуровневой архитектуры, типичной для приложений ASP.NET Core, и организован в несколько проектов в рамках одного решения .NET (Musnews.sln):

Musnews.Server: это основной проект веб-приложения ASP.NET Core, выполняющий функции как уровня приложения, так и уровня представления. Он содержит:
Контроллеры: Предоставляют RESTful API для операций, связанных с музыкой, пользователями и плейлистами.
Сервисы: Реализуют бизнес-логику (например, AuthService) и взаимодействуют с репозиторием.

Musnews.Data: является уровнем доступа к данным, отвечающим за взаимодействие с базой данных. Он включает в себя Entity Framework Core AppDbContext, интерфейс репозитория (ITrackRepository), реализацию репозитория (TrackRepository) и скрипты миграции базы данных.

Musnews.Models: выступает в качестве уровня предметной области, определяя основные сущности (например, Пользователь, Трек, Плейлист) и объекты передачи данных (DTO), используемые во всем приложении.

Musnews.Tests: отдельный проект, посвященный модульному тестированию, обеспечивающему функциональность и надежность основных компонентов, особенно уровня данных.

Основной пользовательский интерфейс обеспечивается статическими HTML-файлами и ресурсами, размещаемыми в каталоге wwwroot, а также дополненными Bootstrap и jQuery.
